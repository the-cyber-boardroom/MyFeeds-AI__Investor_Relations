## Executive Context

MyFeeds.AI is a GenAI-powered personalized news platform initially focused on cybersecurity. The platform uses semantic knowledge graphs and LLMs to solve information overload by delivering "the right information at the right time to the right person" with complete provenance and transparency.

**Key Differentiators:**
- Semantic graph technology with full provenance (not a black box)
- Multi-persona delivery from single content pool
- Revenue-sharing model with content publishers
- Open-source, explainable AI architecture
- Lean serverless architecture (profitable per-user from day one)

**Research Insights Applied:** Based on analysis of successful angel/seed pitch decks from Y Combinator, Sequoia, cybersecurity startups (SplxAI $7M, Blueliv €8M, Protecto $4M, Secureframe $56M), and best practices from 2024-2025, this brief incorporates proven patterns:
- Emphasis on problem urgency and market timing
- Clear visual hierarchy with minimal text
- Traction/validation prominent early
- Team credibility highlighted
- Explainable competitive moat
- Hearts → Minds → Wallets narrative structure

### SLIDE 7: Technology Deep Dive

**Headline:**
```
Technical Architecture: Lean, Scalable, Open-Source Foundation
```

**Content:**

**Architecture Diagram (Detailed):**

**Layer 1: Data Ingestion**
- Multi-provider RSS/API polling
- Extensible plugin architecture
- Raw data preservation (immutable data lake)

**Layer 2: Processing Pipeline (L.E.T.S. Framework)**
- **Load:** Fetch and store raw feeds (XML, JSON)
- **Extract:** Parse into structured schema (TypeSafe validation)
- **Transform:** 
  - Entity extraction via AWS Comprehend
  - MGraph semantic graph creation
  - LLM prompt generation and caching
  - Persona filtering and matching
- **Save:** Store processed outputs, generate digests, update timelines

**Layer 3: Storage & Delivery**
- S3 data lake with path-based versioning
- CloudFront CDN with cache invalidation
- VirtualStorage for LLM prompt caching
- Provenance metadata at every step

**Layer 4: API Layer**
- FastAPI with Lambda integration (Mangum adapter)
- Public routes (no auth): feeds, personas, public data
- Admin routes (API key): flows, debug, config
- LLM cache routes: prompt retrieval for AI agents

**Technology Stack (Table):**

| Component | Technology | Why Chosen |
|-----------|-----------|------------|
| Compute | AWS Lambda | Serverless, pay-per-use, auto-scale |
| Storage | S3 + CloudFront | Cost-effective, global CDN |
| NLP | AWS Comprehend | Managed NER, no ML ops |
| Knowledge Graphs | mgraph-db (OSS) | Semantic reasoning, provenance |
| Type Safety | osbot-utils | Schema validation, fewer bugs |
| LLM Integration | OpenAI/Anthropic APIs | Best-in-class, commodity pricing |

**Cost Model (Critical for investors):**
- **Per-user marginal cost:** €5/month
  - LLM API calls: €2-3
  - AWS services: €1-2
  - Content licensing: €1 (revenue share)
- **Fixed costs:** < €5K/month at current scale
- **Gross margin:** 75-80% at scale

**Scalability Metrics:**
- Current: 1 Lambda instance handles 1,000 articles/day
- With auto-scaling: 50,000 articles/day (no code changes)
- Bottleneck: AWS Comprehend rate limits (100 calls/sec) → addressable with batching

**Open Source Strategy:**
- Core platform: Open source (transparency, community contributions)
- Proprietary: Persona matching algorithms, enterprise features, hosted service
- IP protection: Patents pending on semantic graph + persona mapping

**Visual Elements:**
- Clean architecture diagram (color-coded layers)

## Design & Formatting Guidelines for LLM

**Visual Design Principles:**

1. **Color Palette:**
   - Primary: Deep navy (#0A1128), teal/cyan (#00D9FF) for accents
   - Secondary: Grays (#F5F5F5 backgrounds, #333333 text)
   - Highlights: Green for positive metrics (#00C896), red for urgency (#FF4757)

2. **Typography:**
   - Headlines: Bold, sans-serif (e.g., Montserrat Bold, Roboto Bold), 32-48pt
   - Body text: Sans-serif (e.g., Open Sans, Roboto), 16-20pt
   - Emphasis: Max 15-20 words per bullet point, no walls of text

3. **Layout:**
   - **White space is critical:** Don't overcrowd slides
   - **Hierarchy:** Headlines → Subheads → Bullets → Visuals
   - **Consistency:** Same layout templates across similar slides

4. **Visuals:**
   - Use high-quality icons (avoid clip art)
   - Charts/graphs should be simple and data-focused (no 3D effects)
   - Screenshots: Clean, annotated with callout boxes
   - Architecture diagrams: Color-coded by layer/function

5. **Data Visualization:**
   - Bar charts for comparisons (competitor analysis, fund allocation)
   - Line graphs for trends (revenue growth, market size)
   - Pie charts for distributions (fund allocation, market share)
   - Tables only for dense data that must be compared directly

6. **Professional Polish:**
   - Page numbers on every slide (except title)
   - Consistent footer: "MyFeeds.AI | Seed Investment Deck | Confidential"
   - High-resolution graphics (no pixelation)
   - Proofread for typos and consistency

## Storytelling & Narrative Flow

**Hearts → Minds → Wallets Framework:**

**Light Deck (5 slides):**
- **Slide 1:** Aspirational hook (Hearts)
- **Slides 2-3:** Problem + Solution (Hearts → Minds)
- **Slide 4:** Traction + Market (Minds)
- **Slide 5:** Business model + Ask (Wallets)

**Comprehensive Deck (18 slides):**
- **Slides 1-4:** Emotional urgency (Hearts) - Problem is expensive and worsening
- **Slides 5-12:** Intellectual proof (Minds) - Solution works, team capable, market real
- **Slides 13-18:** Financial logic (Wallets) - Numbers work, returns attractive, risk managed

**Speaking Notes for Presenter:**
- Start with energy and confidence (not arrogance)
- Use pauses after key stats (let data sink in)
- Invite questions at natural breaks (after Problem, after Solution, after Financials)
- Tell stories, not just facts: "One CISO told me..." "When I was running security..."
- End with clarity and urgency: "We're moving fast. Who's ready to join us?"
